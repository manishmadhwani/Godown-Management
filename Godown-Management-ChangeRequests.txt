Godown_Management_RegisterationFlow_1
Allow user to register with you.
User should now be null
User shold not be duplicate
User name and password should not be null, and contact no also shouldn't be null
Once user registers with you send a properMessage, "User sucessfully registered with you", with a Response.ok in response code
Catch exceptions globally, and send proper error messages to the user.
API :- @PostMapping("/register")

Godown_Management_LoginFlow_2
Post registration, allow user to login.
Apply validations on userLoginRequest DTO
Contactno should not be null, password should not be null.
If user does not exist, throw a proper error, saying "User is not registered".
If password doesn't match throw a proper error, saying "Password provided is in-correct"
If no thing fails, return proper ResponseEntity and status code, saying "User has successfully logged in".
API :- @PostMapping("/login")

Godown_Management_GetCityAndStateDetails_3

This will get used in the AddGodown flow, when the user will select the city and state from the dropdown.
The city and state details will get used in forming the godown.
API :- @GetMapping("/fetchAllCities")

Once user logs in he/she should be able to add a godown 
A godown contains, Owner, godown_id(AutoGenerated), name, address, valuation, city details, and entries
At intial, entries will be null, city details should come from the request only.
